# config/

Конфигурационные файлы SOCKS5 прокси-сервера.

## Содержание

| Файл | Назначение |
|---|---|
| [`.env.example`](#env.example) | Шаблон конфигурации с default-значениями |
| [`.env`](#env) | Рабочий конфигурационный файл (сгенерированный или созданный вручную) |

---

### `.env.example`

**Назначение:** Шаблон-образец для генерации рабочего `.env` файла. Содержит значения по умолчанию и не должен редактироваться напрямую (изменения в нём не повлияют на уже существующий `.env`).

**Параметры:**

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ip` | `0.0.0.0` | IP-адрес, на котором слушает прокси |
| `port` | `3128` | Порт прокси-сервера |
| `username` | `secret_user` | Имя пользователя для аутентификации |
| `password` | `secret_user_password` | Пароль для аутентификации |

**Пример содержимого:**
```ini
# SOCKS5 Proxy Configuration

# Network configuration
ip=0.0.0.0
port=3128

# Authentication
username=secret_user
password=secret_user_password
```

---

### `.env`

**Назначение:** Рабочий конфигурационный файл. Создаётся автоматически скриптом [`generate-env.sh`](../scripts/README.md#generate-envsh) или вручную.

**Как создаётся:**
- **Автоматически:** при запуске через `make run`, `make docker-up` или старте Docker-контейнера, если файл отсутствует.
- **Вручную:** копированием из `.env.example` и редактированием.

**Важно:**
- `.env` добавлен в `.gitignore` — не попадает в репозиторий (содержит чувствительные данные).
- При перезапуске контейнера файл **сохраняется** благодаря тому `./config:/app/config` в `docker-compose.yml`.
- Если удалить `.env`, при следующем запуске он будет сгенерирован заново с новыми случайными credentials.

---

## Как настроить

```bash
# 1. Скопировать шаблон
cp config/.env.example config/.env

# 2. Отредактировать под свои нужды
#    - ip, port, username, password

# 3. Запустить прокси
make run
```

Или через Makefile (автоматическая генерация со случайными credentials):
```bash
make generate-env
```
